## 1. Общее описание системы

**Tankatus IS** — информационная система управления IT-проектами, разработанная в виде нативного мобильного приложения для платформы iOS.

Система предназначена для:
- планирования и контроля выполнения IT-проектов;
- управления задачами и подзадачами в рамках проектов;
- мониторинга ключевых показателей эффективности (KPI) по методологии освоенного объёма (EVM);
- распределения нагрузки между участниками команды;
- визуализации хода выполнения работ в форматах Kanban, список и диаграмма Ганта.

Система работает полностью автономно — все данные хранятся в оперативной памяти устройства, сетевое подключение не требуется.

## 4. Аутентификация

### Экран входа

Экран входа (`LoginView`) запрашивает:
- **Email** — адрес электронной почты пользователя;
- **Пароль** — пароль учётной записи.

При нажатии кнопки **«Войти»** система:
1. Проверяет, что поля не пустые.
2. Ищет пользователя по email в локальном хранилище.
3. Проверяет пароль (`Admin123!` для всех тестовых учётных записей).
4. При успехе — сохраняет токен сессии в `UserDefaults` и переводит пользователя на главный экран.
5. При ошибке — отображает сообщение об ошибке.

### Сохранение сессии

Токен сессии сохраняется в `UserDefaults` в формате `local_<userId>`. При повторном запуске приложения сессия восстанавливается автоматически — пользователю не нужно входить заново.

### Выход из системы

Кнопка **«Выйти»** в разделе «Профиль» очищает токен и возвращает пользователя на экран входа.

## 5. Модуль «Дашборд»

Главный экран системы. Обеспечивает быстрый обзор текущего состояния проектов и задач.

### Состав экрана

#### Сводная статистика (4 карточки)
| Карточка | Описание |
|---|---|
| Всего проектов | Общее количество проектов, доступных пользователю |
| Активных проектов | Проекты со статусом «Активный» |
| Задач в работе | Задачи со статусом «В работе» |
| Просроченных задач | Задачи, у которых дата окончания прошла и статус ≠ «Готово» |

#### Активные проекты
Список из до 5 активных и планируемых проектов. По нажатию осуществляется переход к детальному виду проекта. Для каждого проекта отображается:
- название и описание;
- прогресс выполнения (прогрессбар, %);
- статус и приоритет (цветные бейджи);
- имя менеджера.

#### Диаграмма задач по статусам
Секторная диаграмма (`Chart`), отображающая распределение всех задач по статусам:
- К выполнению (todo)
- В работе (in_progress)
- На ревью (review)
- Готово (done)

## 6. Модуль «Проекты»

### Список проектов

Экран отображает все проекты, доступные текущему пользователю:
- **Администратор / Менеджер** — видят все проекты.
- **Сотрудник** — видит только проекты, в которых является участником.

#### Функции списка проектов

**Поиск** — поле поиска в верхней части экрана. Фильтрация по названию проекта в реальном времени.

**Фильтрация по статусу** — меню с фильтрами:
- Все
- Планирование
- Активные
- На паузе
- Завершённые

**Создание проекта** (только Администратор / Менеджер) — кнопка `+` в тулбаре открывает форму создания.

**Карточка проекта** содержит:
- Название и описание
- Бейдж приоритета (Низкий / Средний / Высокий / Критический)
- Прогрессбар (выполненные задачи / всего)
- Счётчик задач `X/Y задач`
- Бейдж статуса
- Имя менеджера
- Дата окончания (красная, если просрочена)

**Редактирование / Удаление проекта** (только Администратор / Менеджер):
- Свайп влево на карточке — кнопки «Изменить» (оранжевая) и «Удалить» (красная).
- Долгое нажатие — контекстное меню с теми же действиями.

### Статусы проекта

| Статус | Описание |
|---|---|
| `planning` | Планирование |
| `active` | Активный, ведётся работа |
| `on_hold` | На паузе |
| `completed` | Завершён |

### Приоритеты проекта

| Приоритет | Описание |
|---|---|
| `low` | Низкий |
| `medium` | Средний |
| `high` | Высокий |
| `critical` | Критический |

### Форма создания / редактирования проекта

Поля формы:
| Поле | Тип | Обязательное |
|---|---|---|
| Название | Текст | Да |
| Описание | Многострочный текст | Нет |
| Статус | Выбор из списка | Да |
| Приоритет | Выбор из списка | Да |
| Бюджет (руб.) | Числовое поле | Нет |
| Дата начала | Выбор даты | Нет |
| Дата окончания | Выбор даты | Нет |

## 7. Модуль «Детальный вид проекта»

Открывается при нажатии на проект в списке. Содержит четыре вкладки.

### Шапка проекта

Всегда отображается над вкладками:
- Бейджи статуса и приоритета
- Дата окончания (с предупреждением о просрочке)
- 4 мини-статистики: Задач / Готово / Просрочено / Прогресс %
- Прогрессбар проекта

### Вкладка «Kanban»

Горизонтальная прокрутка. 4 колонки по статусам задач:

| Колонка | Статус |
|---|---|
| К выполнению | `todo` |
| В работе | `in_progress` |
| На ревью | `review` |
| Готово | `done` |

Каждая **карточка задачи** в Kanban содержит:
- Название задачи
- Прогресс подзадач (если есть): `X/Y` + мини-прогрессбар
- Имя исполнителя
- Бейдж приоритета
- Срок выполнения (красный при просрочке)
- Меню `⋯`:
  - «Редактировать» (Администратор / Менеджер)
  - «Добавить подзадачу» (Администратор / Менеджер)
  - «Удалить» (Администратор / Менеджер)
- Меню «Статус» для быстрой смены (В работе / На ревью / Готово)

### Вкладка «Список»

Иерархическое дерево задач с неограниченной вложенностью.

**Структура отображения:**
- Задачи верхнего уровня — крупный текст, тёмный фон, тень.
- Подзадачи — отступ 20 pt влево, вертикальная синяя линия слева для обозначения уровня.
- Кнопка `▶/▼` для сворачивания / разворачивания ветки.

**Для каждой задачи отображается:**
- Название
- Описание (если есть, одна строка)
- Бейдж статуса + бейдж приоритета
- Срок выполнения
- Имя исполнителя
- Прогресс потомков: `X/Y подзадач` + прогрессбар (если есть дочерние задачи)
- Меню `⋯` с действиями (в зависимости от роли):
  - «Редактировать»
  - «Добавить подзадачу»
  - «Изменить статус» (подменю)
  - «Удалить»

### Вкладка «Гант»

Диаграмма Ганта (`GanttView`) — горизонтальная прокрутка с временной шкалой. Задачи отображаются в виде горизонтальных полос с датами начала и окончания.

### Вкладка «Участники»

Список участников проекта. Для каждого участника:
- Аватар с инициалами
- Полное имя
- Email
- Отдел
- Роль в проекте (менеджер / участник)

### Создание задачи

Кнопка `+` в правом верхнем углу (только Администратор / Менеджер) открывает форму создания новой задачи верхнего уровня.

## 8. Модуль «Задачи»

### «Мои задачи»

Отдельная вкладка в нижней панели навигации. Показывает все задачи, назначенные текущему пользователю, независимо от проекта.

Для каждой задачи отображается:
- Название и статус
- Приоритет
- Название проекта
- Срок выполнения
- Возможность изменить статус

### Иерархическая структура задач (WBS)

Система поддерживает структуру разбивки работ (Work Breakdown Structure) с **неограниченной глубиной вложенности** подзадач.

**Принцип работы:**
- Каждая задача может иметь любое количество дочерних задач (подзадач).
- Каждая подзадача может, в свою очередь, иметь свои подзадачи.
- Вложенность не ограничена.
- Подзадача является **полноценной задачей** — имеет все те же атрибуты.

**Агрегация прогресса:**
- Прогресс родительской задачи вычисляется как доля выполненных потомков на всех уровнях.
- Формула: `doneCount / totalCount`, где `totalCount` — все потомки рекурсивно.

### Форма создания / редактирования задачи

Единая форма `TaskFormView` используется для:
- Создания задачи верхнего уровня
- Создания подзадачи (в шапке формы отображается «Подзадача для: [название родителя]»)
- Редактирования существующей задачи

Поля формы:

| Поле | Тип | Обязательное |
|---|---|---|
| Название | Текст | Да |
| Описание | Многострочный текст | Нет |
| Статус | Выбор из списка | Да |
| Приоритет | Выбор из списка | Да |
| Оценка трудозатрат (часы) | Числовое поле | Нет |
| Исполнитель | Выбор из участников проекта | Нет |
| Этап | Выбор из этапов проекта | Нет |
| Срок выполнения | Выбор даты | Нет |

### Статусы задач

| Статус | Описание |
|---|---|
| `todo` | К выполнению |
| `in_progress` | В работе |
| `review` | На ревью |
| `done` | Готово |

### Приоритеты задач

| Приоритет | Описание |
|---|---|
| `low` | Низкий |
| `medium` | Средний |
| `high` | Высокий |
| `critical` | Критический |

### Этапы проекта

Задачи привязываются к этапам проекта. Этапы имеют собственный порядок и статус. Для проекта «Разработка интернет-магазина v2» определены три этапа:

| Этап | Статус | Период |
|---|---|---|
| Анализ требований | Завершён | 01.01–31.01.2026 |
| Дизайн и прототипирование | Завершён | 01.02–28.02.2026 |
| Разработка | В работе | 01.03–31.05.2026 |

## 9. Модуль «KPI / EVM»

Раздел предназначен для мониторинга эффективности выполнения проектов по методологии **Earned Value Management (EVM)** — управление освоенным объёмом.

### Выбор проекта

В верхней части экрана расположен выпадающий список для выбора проекта. Отображаются проекты со статусами «Активный» и «Планирование».

### KPI-показатели (4 карточки)

| Показатель | Формула | Норма |
|---|---|---|
| **SPI** — Индекс выполнения расписания | EV / PV | ≥ 1.0 — зелёный, < 1.0 — красный |
| **CPI** — Индекс выполнения стоимости | EV / AC | ≥ 1.0 — зелёный, < 1.0 — красный |
| **CV** — Отклонение по стоимости | EV − AC | ≥ 0 — зелёный, < 0 — красный |
| **SV** — Отклонение по расписанию | EV − PV | ≥ 0 — зелёный, < 0 — красный |

### EVM-график

Линейный график с тремя кривыми по временным снимкам:
- **PV** (Planed Value) — плановый объём, синяя линия
- **EV** (Earned Value) — освоенный объём, зелёная линия
- **AC** (Actual Cost) — фактическая стоимость, оранжевая линия

По оси X — дата снимка, по оси Y — значение в рублях.

### Прогнозные показатели

| Показатель | Формула | Описание |
|---|---|---|
| **EAC** — Прогнозная стоимость | BAC / CPI | Ожидаемая итоговая стоимость проекта |
| **ETC** — Стоимость оставшихся работ | EAC − AC | Сколько ещё потребуется вложить |
| **BAC** — Бюджет по завершении | Задан при создании | Исходный утверждённый бюджет |

### Снимки KPI

Данные для графиков хранятся в виде снимков (snapshot) по датам. В системе предзаполнены три снимка для демонстрационного проекта:

| Дата | PV | EV | AC | SPI | CPI |
|---|---|---|---|---|---|
| 01.02.2026 | 25 000 | 23 000 | 24 000 | 0.92 | 0.96 |
| 01.03.2026 | 50 000 | 48 000 | 47 500 | 0.96 | 1.01 |
| 01.04.2026 | 75 000 | 72 000 | 71 000 | 0.96 | 1.01 |

### TaskNode — Узел дерева задач

Рекурсивная структура для построения иерархии:

```
TaskNode
├── task: ProjectTask        — данные задачи
├── children: [TaskNode]     — дочерние узлы (рекурсивно)
├── allDescendants           — все потомки на всех уровнях
├── doneCount                — количество выполненных потомков
├── totalCount               — общее количество потомков
└── subtaskProgress          — прогресс: doneCount / totalCount
```
